    병렬 개발의 핵심은 shared 폴더입니다. broker와 trader가 소통할 데이터 형식을 shared/models.py에 먼저 정의해두고, AI에게 **"이 모델에 맞춰서 각 모듈을 구현해"**라고 지시하면 두 사람이 동시에 개발해도 나중에 합칠 때 충돌이 거의 없습니다.

  3. 커밋(Commit) 단위 분리:
    AI에게 한 번에 모든 폴더를 수정하게 하지 마세요.

    • "Claude, 지금은 broker 모듈의 실시간 호가 수집 기능만 구현해줘."

    • (작업 완료 후 검토 및 커밋)

    • "이제 data-center에서 이 데이터를 저장하는 기능을 만들어줘."
      이렇게 모듈별로 작업을 쪼개서 진행하는 것이 훨씬 안정적입니다.

  4. 문서화(Documentation) 유지:
    각 폴더(모듈)마다 간단한 README.md를 두어 해당 모듈의 역할과 입출력을 적어두세요. Claude Code는 이 파일을 읽고 해당 모듈의 역할을 즉시 이해하여 더 나은 제안을 해줍니다.

결론

"단일 저장소 + 독립 폴더" 방식은 AI 도구를 사용하기에 가장 최적화된 환경입니다. 모듈 간의 '약속(Interface)'만 처음에 잘 정의해 두신다면, AI와 함께 아주 빠르게 프로토타입을 완성하실 수 있을 것입니다. 어느 모듈의 폴더 구조부터 먼저 설계해볼까요?
